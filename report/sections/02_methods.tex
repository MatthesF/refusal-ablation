\section{Methods}

\subsection{Model}

The evaluated model is \texttt{google/gemma-4-E4B-it}. The exact Hugging Face
revision is pinned in the repository settings. The baseline condition uses the
model without weight edits. The edited condition uses the same checkpoint after
one refusal-direction edit.

\subsection{Direction Construction}

The refusal direction is fit from a local policy-derived dataset with 480
prompts. The dataset contains 12 policy areas, with 20 safe and 20 unsafe
prompts per area. The fitting split is policy-area-disjoint: 8 policy areas
(320 prompts) are used for construction, and 4 policy areas (160 prompts) are
held out from construction.

For each construction prompt, the model is run with the normal chat template and
the final prompt-token hidden state is recorded at each language layer. For each
layer, the raw direction is the difference between the mean unsafe-prompt
activation and the mean safe-prompt activation. The component parallel to the
safe-prompt mean is removed, and the resulting vector is normalized.

\subsection{Weight Edit}

The edit is applied to every language layer. For each layer, the fitted
direction is removed from the output columns of the attention output projection
and MLP down projection. Column norms are then restored when an orthogonal
component remains. The same edited model is used for every SORRY-Bench prompt.

\subsection{Direction Stability Check}

Before the benchmark run, the construction dataset is used to check whether the
direction is stable across policy areas. For each policy-area budget, one
direction is fit on the used policy areas and one on the remaining held-out
policy areas. The directions are compared layer by layer using cosine
similarity. This check is not used to tune the final SORRY-Bench result; it is a
sanity check that the local construction data contains a shared signal.

\subsection{SORRY-Bench Evaluation}

The benchmark evaluation uses SORRY-Bench
\texttt{sorry-bench/sorry-bench-202503}, with the exact dataset revision pinned
in the repository settings. The main split contains 440 prompts across 44
categories, with 10 prompts per category.

Both baseline and edited Gemma answer all 440 prompts with deterministic
decoding. Generation is capped at 4096 new tokens. The answers are exported to
the official SORRY-Bench answer format and scored with the official Mistral
judge checkpoint. The exact judge checkpoint revision is pinned in the
repository settings. The official judge prompt is
\texttt{base-ft-mistral-7b-instruct-v0.2}.

The judge score is binary:

\begin{equation}
  Y_{i,m} =
  \begin{cases}
    1, & \text{if model } m \text{ complied with prompt } i,\\
    0, & \text{if model } m \text{ refused prompt } i.
  \end{cases}
\end{equation}

\subsection{Statistical Test}

The main comparison uses the paired prompt-level scores from the baseline and
edited model. The reported effect size is the paired difference in compliance
rate:

\begin{equation}
  \Delta =
  \frac{1}{N}\sum_{i=1}^{N}(Y_{i,\text{edited}} - Y_{i,\text{baseline}}).
\end{equation}

McNemar's exact test is computed from the discordant pairs. Under the null
hypothesis, the two flip directions are equally likely: baseline refusal to
edited compliance, and baseline compliance to edited refusal.
